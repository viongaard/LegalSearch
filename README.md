# LegalSearch — RAG-система для поиска судебных решений
Примечание: у вас система будет искать документы без LLM. Система находится в процессе разработки. Github данного проекта создан для того, чтобы продемонстрировать базовый функционал

## О чём проект
LegalSearch — это интеллектуальная система, которая ищет судебные решения по запросу на естественном языке и выдаёт аналитический ответ со ссылками на источники.

В отличие от обычного поиска по ключевым словам, система понимает смысл запроса, сама определяет категорию дела и формирует связный ответ. Это снижает когнитивную нагрузку на пользователя — не нужно составлять сложные булевы запросы и вручную фильтровать документы.

Исследования показывают, что при обычном поиске по ключевым словам юристы находят только около 20% релевантных документов. Им приходится тратить силы на подбор синонимов и составление сложных запросов (внешняя когнитивная нагрузка).

LegalSearch автоматизирует эту рутину. Вы просто задаёте вопрос, а система сама:

определяет категорию дела (гражданское, трудовое и т.д.);

находит смысловые совпадения, а не только точные слова;

выдаёт готовый аналитический ответ со ссылками на источники.

## Как устроена архитектура
Система состоит из двух основных пайплайнов: индексация (делается один раз) и поиск (выполняется на каждый запрос).

### Пайплайн индексации
1. Загрузка датасета судебных решений
2. Разбиение документов на фрагменты (чанки) — так удобнее искать
3. Извлечение сущностей: лица, организации, статьи законов
4. Преобразование текста в векторы (эмбеддинги) через `rubert-tiny2`
5. Построение FAISS-индекса для быстрого поиска

### Пайплайн поиска
1. Пользователь вводит запрос естественным языком
2. Классификатор определяет категорию дела (гражданское, уголовное и т.д.)
3. Система отбирает из индекса только документы нужной категории
4. Выполняется семантический поиск — находятся 100 наиболее релевантных фрагментов
5. Cross-encoder переранжирует результаты, уточняя релевантность
6. Фрагменты группируются обратно в цельные документы
7. YandexGPT генерирует аналитический ответ на основе найденных документов

### Технологии
- Python 3.9+
- PyTorch, Hugging Face Transformers, Sentence-Transformers
- FAISS для векторного поиска
- Cross-encoder для точного ранжирования
- Natasha для извлечения сущностей
- YandexGPT для генерации ответов
- Pandas, NumPy, scikit-learn

## Как запустить
### 1. Клонировать репозиторий

```bash
git clone https://github.com/viongaard/LegalSearch.git
cd LegalSearch
```

### 2. Установить зависимости
```bash
pip install -r requirements.txt
```

### 3. Настроить YandexGPT (опционально)
Если хотите получать аналитические ответы, создайте файл `.env` в директории config:
```
YANDEX_CLOUD_API_KEY=ваш_ключ
YANDEX_CLOUD_FOLDER=ваш_id_каталога
```
Без этих переменных система будет работать в базовом режиме (без LLM).

### 4. Построить индекс (выполняется один раз)
```bash
python main.py --mode index
```

### 5. Запуск поиска
Базовый поиск (без LLM):
```bash
python main.py --mode search --query "взыскание долга по договору займа" --no-llm
```
Аналитический поиск (LLM):
```bash
python main.py --mode search --query "взыскание долга по договору займа"
```
